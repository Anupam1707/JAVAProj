\section*{Abstract}
This project presents a Smart City Traffic Simulator designed to model traffic flow and signal control at a four-way intersection. The main purpose is to demonstrate object-oriented design and event-driven programming through a practical Java application. The system is implemented using Java and Java Swing for the graphical interface. Core features include adaptive traffic signal rotation, emergency vehicle override for ambulances, lane-wise vehicle tracking, real-time simulation statistics, pause/resume control, and speed adjustment. The project supports file handling through a persistent logging component that stores simulation output for review. Overall, the simulator provides an educational and extensible platform for understanding how software can represent real traffic management logic while applying key OOP concepts such as abstraction, inheritance, polymorphism, interfaces, and exception handling.

\section{Introduction}
Rapid urbanization has increased traffic congestion, travel delays, and emergency response times in modern cities. Traditional fixed-time traffic systems cannot always adapt to changing traffic conditions, which makes intersection management inefficient. This project addresses that problem by building a Smart City Traffic Simulator that models traffic light behavior, vehicle movement, and emergency vehicle prioritization at an intersection. The application is important because it demonstrates how software-based traffic logic can improve flow, reduce waiting time, and support safer road usage. It also serves as a practical academic project for understanding object-oriented programming, GUI design, and event-driven systems using Java.

\section{Implementation}
The application was implemented in Java using a modular OOP architecture. The GUI was developed with Swing components such as \texttt{JFrame}, \texttt{JPanel}, \texttt{JButton}, \texttt{JLabel}, and \texttt{JSlider}. User actions (for example: spawning traffic, toggling smart mode, pause/resume, and viewing logs) are handled through event listeners attached to buttons and controls. The simulation runs using timers: one timer updates vehicle animation and another updates traffic-light logic each second. Vehicle behavior is implemented polymorphically through the abstract \texttt{Vehicle} class and its subclasses. Emergency handling is implemented through the \texttt{EmergencyVehicle} interface and ambulance override logic in \texttt{TrafficController}. File handling is implemented via \texttt{SimulatorLogger}, which writes timestamped logs to disk and also maintains an in-memory buffer for GUI log viewing. No external database is used in the current version.

\section{Results and Discussion}
The developed simulator produced stable and expected results across all planned test cases. Signal transitions occurred correctly, vehicles moved according to lane permissions, and emergency vehicles received immediate priority through override logic. The GUI improved visibility of internal system states, making behavior easy to verify during execution. Based on testing outcomes, the project objectives were achieved: the system models intersection control effectively, demonstrates core Java OOP concepts, and provides a usable interface for simulation. The results indicate that even a simplified traffic model can meaningfully represent real-world control challenges and provide a foundation for advanced smart-traffic research.

\section{Limitations}
\begin{itemize}
    \item Simplified single-intersection model without city-wide network behavior
    \item Limited adaptive intelligence (no ML-based dynamic optimization)
    \item Basic persistence using files only; no full database integration
    \item Vehicle behavior does not include advanced factors like accidents or lane blocking
\end{itemize}

\section{Future Enhancements}
\begin{itemize}
    \item Integrate multiple intersections with coordinated signal control
    \item Add AI/ML-based adaptive signal timing based on live traffic density
    \item Introduce complete database support for long-term analytics
    \item Enhance GUI with charts, heatmaps, and real-time performance metrics
\end{itemize}

\section{Conclusion}
This work successfully implemented a Smart City Traffic Simulator using Java and Swing. The project combines object-oriented design, GUI interaction, event handling, and file-based logging into a functional academic application. Key learning outcomes include class hierarchy design, abstraction through abstract classes and interfaces, runtime polymorphism, timer-driven simulation modeling, and practical GUI development with Swing. The completed system meets its core objectives: simulating intersection behavior, supporting adaptive signal control, and prioritizing emergency vehicles. It provides a strong and extensible foundation for future smart-traffic enhancements.
